## 📑 THE SHIFT: FROM Context Bloat TO Modular Playbooks

As analyzed in the trending open-source repository configurations, the classic approach of dumping massive, monolithic instructions into a global LLM prompt is highly inefficient. It:
1.  **Increases Token Burn**: Artificially inflates prompt footprints, multiplying API usage costs.
2.  **Degrades Focus**: Floods the model's active attention window with instructions irrelevant to the specific sub-task (e.g., teaching data-ink principles while checking SQLite WAL transactions).
3.  **Causes Reasoning Drift**: Prompts the model to balance conflicting guidelines arbitrary (e.g., trading off security parameters for swift styling).

To solve this, the **Age Republic** adopts the **Modular Playbook Standard**. We structure instructions as local, independent, and state-triggered markdown modules in `.age-skills/trending_skills/`. 

These modules remain completely dormant until the orchestrator flags a matching context seam, dynamically opening a brief execution branch to enforce strict policies, and dropping them once compilation completes.
